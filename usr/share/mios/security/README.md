# MiOS agent egress firewall (#54)

Zero-trust agent federation needs an **outbound** control: a compromised or
misled agent must not be able to exfiltrate to arbitrary internet hosts. The
correct layer is the OS — **nftables**, scoped to the agent's uid. An app-level
hook would be incomplete (the orchestrator constructs httpx clients ad-hoc), so
this is enforced below the app.

## `egress.nft`

Generated by
[`tools/generate-egress-firewall.py`](../../../../tools/generate-egress-firewall.py)
from `mios.toml [security.egress]` + the agent-pipe service `User=` (SSOT — the
uid is not hardcoded). The ruleset is **uid-scoped to the agent**:

- All traffic from **other users** is accepted untouched — so `web_search` keeps
  working (the agent reaches searxng over loopback; searxng, a different uid,
  reaches the internet).
- For the **agent user**, these are always allowed: loopback, tailnet
  (`100.64.0.0/10`), the local WSL gateway (`172.16.0.0/12`), and any extra
  destinations in `[security.egress].allow`.
- Everything else the agent tries to reach is handled per **mode**.

## Modes (`[security.egress].mode`)

| mode | behaviour |
|---|---|
| `off` (default) | generator emits a **no-op** ruleset; applying it changes nothing |
| `audit` | the agent's non-allowed external egress is **logged**, then accepted (observe what enforce *would* block, with zero risk) |
| `enforce` | the agent's non-allowed external egress is **logged + dropped** |

## Applying it (operator step — nothing here is auto-applied)

```bash
# regenerate after editing [security.egress] (run where the MiOS stack lives)
tools/generate-egress-firewall.py

# review, then apply / remove
sudo nft -f usr/share/mios/security/egress.nft
sudo nft delete table inet mios_egress      # to remove

# make it boot-durable: load egress.nft from your nftables include path
```

Recommended rollout: `audit` first (watch `journalctl -k | grep mios-egress-audit`
for a few days to confirm nothing legitimate is caught), widen
`[security.egress].allow` as needed, then switch to `enforce`. Validated with
`nft -c -f egress.nft` for all modes.

# Agent mTLS PKI (#54)

The ed25519 message principal (#60) authenticates a *delegation*; **mutual TLS**
authenticates the *transport* between A2A peers. The PKI is minted by
[`tools/provision-agent-mtls.py`](../../../../tools/provision-agent-mtls.py):

```bash
# operator step (writes secrets; run where the certs should live)
tools/provision-agent-mtls.py        # -> /etc/mios/mtls/{ca,agent}.{crt,key}
```

- A **self-signed local CA** (the standard self-hosted default) signs an **agent
  leaf** cert valid for both `clientAuth` and `serverAuth`. To use an existing
  **org PKI** instead, point `[security.mtls].ca_file/cert_file/key_file` at it.
- Re-running **reuses the CA** (so peer trust you've already exchanged stays
  valid) and re-issues the leaf. Keys are written `0600`. Certs are **secrets**
  (per-host, time-stamped) and are **not committed**.

**Establishing peer trust:** exchange `ca.crt` files between MiOS nodes (each
node trusts the others' CA), so `agent.crt` presented by a peer validates.

**Enforcing it (deployment, not app code):** MiOS terminates TLS at the reverse
proxy (the same pattern as the pretty-URL `tailscale serve --tls-terminated-tcp`
fronting). Configure the proxy in front of the A2A endpoint to **require + verify
client certs** against the peer CAs, e.g.:

```
# nginx sketch in front of the /a2a endpoint
ssl_client_certificate /etc/mios/mtls/peer-cas.crt;   # concatenated peer CA certs
ssl_verify_client on;
proxy_pass http://127.0.0.1:8640;
```

`[security.mtls].enable` is the advisory flag for that deployment. With no certs
provisioned, A2A stays plain (current single-node behaviour) — nothing here is
auto-applied.
