# FOSS Upstream-Scout Report — MiOS (mid-2026)

MiOS's FOSS upstream is unusually healthy and converging on exactly MiOS's architecture: the immutable core (bootc) shipped its composefs+fs-verity backend and Logically Bound Images in the v1.16.x line ([bootc](https://github.com/bootc-dev/bootc)), while the peer field (Bluefin/Bluespeed, elizaOS) has independently landed the same "Fedora bootc + Podman + user-owned OpenAI /v1 endpoint" thesis MiOS is built on. The one structural disruption to absorb now: `bootc-image-builder` was archived read-only on 2026-06-18 and folded into [osbuild/image-builder](https://github.com/osbuild/image-builder) — MiOS build scripts pointing at the old repo are on a dead path. The AI-serving and agent-protocol layers have matured into drop-in Quadlet candidates (vLLM, llama.cpp, ContextForge, Agentgateway), each speaking the OpenAI /v1 surface MiOS fronts via Hermes :8642. The desktop-shell lane finally has a credible engine — Quickshell is now the de-facto substrate under every major 2026 Wayland shell — making the liquid-glass north star projectable from `mios.toml` rather than hand-built.

## Immutable / bootc OS core

1. **[bootc](https://github.com/bootc-dev/bootc)** — Apache-2.0 OR MIT (dual). *What's new:* v1.16.4 (2026-07-15); the 2025–26 work pushes a composefs-native backend with per-file fs-verity verification (composefs manifest as GC root) and Logically Bound Images that pin app containers to the host image so they land before reboot ([bootc](https://github.com/bootc-dev/bootc)). **MiOS action — adopt_now:** this is MiOS's literal OS engine; adopt the composefs+fs-verity backend for a sealed verified root and use Logically Bound Images to bind the /v1 AI plane + agent-pipe sidecar Quadlets to the host image so they land atomically on `bootc switch`, replacing drift-prone post-boot pulls.

2. **[bootc-image-builder → osbuild/image-builder](https://github.com/osbuild/bootc-image-builder)** — Apache-2.0. *What's new:* standalone repo **archived read-only 2026-06-18**; capability now lives in [osbuild/image-builder](https://github.com/osbuild/image-builder) (Fedora's official build infra). Converts a bootc OCI image into qcow2/ami/vmdk/raw/vhd/gce and ISO (bootc-installer + anaconda-iso) artifacts ([osbuild/bootc-image-builder](https://github.com/osbuild/bootc-image-builder)). **MiOS action — adopt_now:** this is the bridge from MiOS's OCI image to Ventoy-droppable media; migrate the MiOS build pipeline off the archived repo to `osbuild/image-builder` and record it in mios-cat-deploy-readiness (bare-metal leg).

3. **[Universal Blue uCore / ucore-hci](https://github.com/ublue-os/ucore)** — Apache-2.0. *What's new:* rolling daily builds tracking Fedora 44, LTS kernel 6.18, ARM64/aarch64 manifests added 2025-11; `-hci` variant adds socket-activated libvirtd + cockpit-machines/virt-install ([ublue-os/ucore](https://github.com/ublue-os/ucore)). **MiOS action — adopt_now:** MiOS builds FROM uCore; ucore-hci's socket-activated libvirtd is the exact pattern for hosting Linux+Windows guests beside the immutable host, and the new aarch64 manifests open a multi-arch lane. Watch its Fedora bumps to reconcile the 3-way `mios.toml` version drift (0.3.0/0.2.4).

4. **[Bluefin](https://github.com/ublue-os/bluefin)** — Apache-2.0. *What's new:* stable-20260714 (Fedora 44); Spring 2026 moved fully to composefs+fs-verity ("goodbye ostree"), added TPM-backed passwordless unlock via systemd-cryptenroll, went UEFI-only, and refactored Aurora+Bluefin into shared common OCI layers ([ublue-os/bluefin](https://github.com/ublue-os/bluefin)). **MiOS action — watch:** the reference implementation of a composefs+fs-verity+TPM-sealed bootc desktop; mine its shared-OCI-layer factoring and desktop packaging as a model for the Hyprland/liquid-glass lane. Reference, not a base.

## Universal bootable USB / recovery (MiOS-Cat deploy plane)

1. **[Ventoy](https://github.com/ventoy/Ventoy)** — GPL-3.0. *What's new:* v1.1.16 (2026-06-25); the 1.1.14–1.1.16 run swapped the Secure Boot shim to a Rocky Linux shim signed with both UEFI CA 2011 and 2023, so first boot on Secure Boot hardware now requires enrolling `ENROLL_THIS_KEY` via MokManager ([ventoy/Ventoy](https://github.com/ventoy/Ventoy)). **MiOS action — adopt_now:** this is the boot engine already under MiOS-Cat; pin a known-good release as a supply-chain decision, project `ventoy.json` from `mios.toml`, and wire MiOS's own keys into the shim+GRUB+MOK chain (avoid Ventoy's "fully bypass secure boot" policy) to keep verified boot in line with MiOS's chain-of-trust.

2. **[Rescuezilla](https://github.com/rescuezilla/rescuezilla)** — GPL-3.0. *What's new:* v2.6.2 (2026-05-16), Ubuntu 24.04 rolling toward 26.04 LTS, wraps partclone 0.3.47 for used-space imaging; produces Clonezilla-interoperable images; v2.6.2 fixed a blank-error bug blocking auto-restart/shutdown ([rescuezilla/rescuezilla](https://github.com/rescuezilla/rescuezilla)). **MiOS action — watch:** drop-in loopback ISO for MiOS-Cat's recovery menu giving bare-metal image/backup/restore of bootc host disks and hosted VMs; Clonezilla-format is a natural field capture format. Track the slated ARM64 build since MiOS targets ARM64 firmware.

3. **[SystemRescue](https://gitlab.com/systemrescue/systemrescue-sources)** — GPL-3.0-or-later (mixed, Arch-based). *What's new:* v13.01 (2026-06-06) on kernel 6.18.34; added a `checksum` boot option verifying image integrity under copytoram, plus b3sum/reflector and bcachefs-tools/module 1.38.3 ([systemrescue-sources](https://gitlab.com/systemrescue/systemrescue-sources)). **MiOS action — watch:** bundle as a MiOS-Cat rescue loopback entry; its copytoram+checksum path mirrors MiOS's SBOM/verified-boot ethos and is a concrete reference for checksum-verifying MiOS-Cat payloads before RAM-loading.

4. **[netboot.xyz](https://github.com/netbootxyz/netboot.xyz)** — Apache-2.0. *What's new:* v3.0.2 (2026-05-06); the 3.x bump refreshed signed bootloader assets for UEFI Secure Boot after a code-signing cert update; self-hostable via Docker/Ansible generating BIOS+UEFI iPXE disks ([netbootxyz/netboot.xyz](https://github.com/netbootxyz/netboot.xyz)). **MiOS action — reference:** the network counterpart to the USB plane — self-host a mirror as a Quadlet-managed container to iPXE-boot the MiOS bootc installer across a fleet; its signed-iPXE Secure Boot path is a model for MiOS's own signed network-install chain projected from `mios.toml`.

## Offline image-based install / deployment (registry-free bare-metal leg)

1. **[bootc — install engine](https://github.com/containers/bootc)** — Apache-2.0 OR MIT. *What's new:* v1.16.4 (2026-07-15); `bootc install to-disk`/`to-filesystem` writes the container to bare metal, and fully-disconnected installs run via `--transport OCI`/`oci-archive` against a container copied to local storage (e.g. a USB OCI dir) with zero registry access ([containers/bootc](https://github.com/containers/bootc)). **MiOS action — adopt_now:** the `bootc switch --transport oci /var/mnt/usb/mios.oci` path is exactly the air-gapped Ventoy flow the deploy-readiness memo flags as missing (tools/install.sh absent); wire `mios.toml` SSOT to emit the install invocation and pin the bootc version in SBOM at build, not by hand.

2. **[osbuild/image-builder — anaconda-iso](https://github.com/osbuild/image-builder)** — Apache-2.0. *What's new:* consumes a bootc container and emits qcow2/raw/ami/vmdk/vhd/gce/PXE plus two ISO flavors; the **anaconda-iso embeds the container content so install needs no network** ([osbuild/image-builder](https://github.com/osbuild/image-builder)). **MiOS action — adopt_now:** this is the "correct BIB installer" the deploy-readiness memo says is disconnected/unbuilt — bake the MiOS OCI image into an embedded-container anaconda-iso and drop it on the Ventoy/MediCat USB for true air-gapped installs. Point build scripts at the new repo.

3. **[ublue-os/ucore — base cadence](https://github.com/ublue-os/ucore)** — Apache-2.0. *What's new:* rolling, tracking Fedora 44 (2026-02), LTS kernel 6.18, aarch64 since 2025-11; installed via CoreOS Ignition + bootc; batteries include Podman/Cockpit/Tailscale/ZFS/optional NVIDIA ([ublue-os/ucore](https://github.com/ublue-os/ucore)). **MiOS action — adopt_now:** its Fedora-44/kernel-6.18 cadence and Ignition/bootc install expectations govern MiOS's `FROM` line and offline-install assumptions; NVIDIA+Tailscale batteries overlap MiOS's driver/networking needs.

4. **[CoreOS Ignition](https://github.com/coreos/ignition)** — Apache-2.0. *What's new:* v2.26.0 (2026-02-23); runs once in initramfs before root mount to partition/format disks, write files + systemd units, and configure users from declarative JSON ([coreos/ignition](https://github.com/coreos/ignition)). **MiOS action — watch:** because MiOS builds FROM uCore/CoreOS, Ignition is the first-boot seed layer — a natural target for projecting `mios.toml` SSOT into install-time config (author Butane YAML, compile to Ignition) so first-boot provisioning stays drift-gated.

## Local AI serving (OpenAI /v1)

1. **[vLLM](https://github.com/vllm-project/vllm)** — Apache-2.0. *What's new:* v0.25.1 (2026-07-14); PagedAttention KV cache + continuous batching, OpenAI-compatible server (Chat/Completions/Embeddings) plus Anthropic Messages and gRPC, structured output via XGrammar, native parallel tool-calling, single-GPU to multi-node TP/PP ([vllm-project/vllm](https://github.com/vllm-project/vllm)). **MiOS action — adopt_now:** primary self-hosted /v1 backend for GPU nodes as a Podman/Quadlet container fronted unchanged by Hermes :8642; `:latest` + SBOM-at-build fits ADR-0003 and the OpenAI surface means zero agent-pipe changes.

2. **[llama.cpp / llama-server](https://github.com/ggml-org/llama.cpp)** — MIT. *What's new:* b10064 (2026-07-17); C/C++ GGUF inference on CPU/consumer GPU, llama-server exposes OpenAI-compatible /v1/chat/completions with grammar/JSON-schema constrained output and speculative decoding, ships as a single static binary ([ggml-org/llama.cpp](https://github.com/ggml-org/llama.cpp)). **MiOS action — adopt_now:** the CPU/consumer-GPU path for heterogeneous MiOS-Cat hardware where vLLM's datacenter assumptions fail; single static binary + GGUF suits the immutable bootc image — expose behind the same /v1 endpoint as a low-resource fallback Quadlet.

3. **[RamaLama](https://github.com/containers/ramalama)** — MIT. *What's new:* v0.23.0 (2026-06-24); containers-org (Red Hat) tool that pulls/serves models as OCI images under Podman with automatic GPU detection, backing serving with llama.cpp or vLLM, exposing OpenAI-style API + web UI ([containers/ramalama](https://github.com/containers/ramalama)). **MiOS action — watch:** architecturally the closest match — models-as-OCI-images under Podman on bootc/uCore is exactly MiOS's Quadlet substrate; study its OCI-model packaging to validate the "model as SBOM-tracked OCI artifact" direction even if not adopted wholesale.

4. **[llama-swap](https://github.com/mostlygeek/llama-swap)** — MIT. *What's new:* v240 (2026-07-15); zero-dependency Go proxy (one binary + one YAML) that hot-loads/unloads backends (llama.cpp, vLLM, TabbyAPI) by model id, fronting /v1/chat/completions, /v1/embeddings, /v1/messages with per-model TTLs and VRAM groups ([mostlygeek/llama-swap](https://github.com/mostlygeek/llama-swap)). **MiOS action — watch:** lets one MiOS /v1 endpoint route to many models and auto-evict under VRAM pressure without touching the agent-pipe; single Go binary + YAML projects cleanly from `mios.toml` as a thin Quadlet in front of vLLM/llama.cpp.

## Agent frameworks & protocols

1. **[IBM ContextForge (mcp-context-forge)](https://github.com/IBM/mcp-context-forge)** — Apache-2.0. *What's new:* v1.0.5 (2026-07-07); unified MCP gateway federating MCP+REST+gRPC behind one endpoint, wrapping non-MCP services as "virtual MCP servers" via JSON-Schema extraction + gRPC reflection, with OpenTelemetry tracing (Phoenix/Jaeger/Zipkin) and centralized auth/rate-limit/audit ([IBM/mcp-context-forge](https://github.com/IBM/mcp-context-forge)). **MiOS action — adopt_now:** ship as one Quadlet fronting every MCP+REST+gRPC tool for the agent-pipe; the virtual-server catalog is a natural `mios.toml` projection target (drift-gated) and its auth/rate-limit/audit chokepoint sits cleanly in front of Hermes :8642.

2. **[Agentgateway (Linux Foundation)](https://github.com/agentgateway/agentgateway)** — Apache-2.0. *What's new:* v1.3.1 (2026-06-22); Rust-primary data plane speaking MCP and A2A natively, exposing a unified OpenAI-compatible API across providers with budget/load-balancing, JWT/API-key/OAuth auth, CEL-based RBAC, TLS, and OpenTelemetry ([agentgateway/agentgateway](https://github.com/agentgateway/agentgateway)). **MiOS action — adopt_now:** a vendor-neutral LF-governed Rust proxy is an ideal single security+observability chokepoint for MiOS's multi-agent council routing in front of /v1; CEL RBAC is another `mios.toml`-projectable, drift-gated surface, and the low binary footprint suits an immutable bootc host. Deploy as a Quadlet.

3. **[MCP Registry](https://github.com/modelcontextprotocol/registry)** — Apache-2.0 (new code); MIT (older contributions); CC-BY-4.0 (docs). *What's new:* v1.8.0 (2026-07-13); curated MCP-server registry on PostgreSQL with seeding, publishing gated by GitHub OAuth/OIDC and DNS/HTTP domain-ownership verification; stateless-HTTP spec RC due 2026-07-28 ([modelcontextprotocol/registry](https://github.com/modelcontextprotocol/registry)). **MiOS action — adopt_now:** self-host a MiOS-scoped instance so the agent plane serves an operator-approved MCP catalog projected from `mios.toml` instead of arbitrary upstream servers; Postgres backing pairs with an existing pgvector/Postgres Quadlet and the DNS/OIDC publish gating maps to MiOS's governed config posture.

4. **[Agent2Agent (A2A)](https://github.com/a2aproject/A2A)** — Apache-2.0. *What's new:* v1.0.1 (2026-05-28); LF-hosted (Google-donated) cross-framework protocol over JSON-RPC 2.0/HTTP(S), Agent Cards for capability discovery, sync + SSE + async push ([a2aproject/A2A](https://github.com/a2aproject/A2A)). **MiOS action — watch:** adopt A2A as the wire standard for council/multi-agent handoff to avoid framework lock-in; both ContextForge and Agentgateway already speak it, so standardizing lets those gateways front the mesh. A MiOS A2A directory (Agent Cards) is a watch-list build.

## Peer / competitor AI-OS projects

1. **[elizaOS](https://github.com/elizaOS/eliza)** — MIT. *What's new:* v2.x line (v2.0.2 reported stable ~July 2026); local-first runtime with an on-device model family, and `packages/os/linux` boots a full Linux desktop from USB (amd64/arm64/riscv64) with Eliza baked in ([elizaOS/eliza](https://github.com/elizaOS/eliza)). **MiOS action — watch:** the closest conceptual twin to MiOS-Cat — a USB-bootable local-first "AI OS" with the agent in the base image; benchmark MiOS's bootc/BIB+Ventoy deliverable and on-device agent-pipe UX directly against `packages/os/linux`, and study how they bake agent+models into the bootable image vs MiOS's Quadlet-composed /v1 plane.

2. **[Bluefin / Bluespeed](https://github.com/ublue-os/bluefin)** — Apache-2.0. *What's new:* stable-20260714 (Fedora 44); "Bluespeed" AI layer ships Ramalama (OpenAI-compatible at :8080), Docker Model Runner (:12434), Alpaca GUI, and Goose agent, with the core principle of a single user-controlled OpenAI-compatible endpoint ([ublue-os/bluefin](https://github.com/ublue-os/bluefin)). **MiOS action — watch:** MiOS's most direct architectural competitor (identical Fedora bootc + podman + ramalama + user-owned /v1 substrate) — note their Ramalama default :8080 is precisely the port MiOS retired for Hermes :8642 (a concrete positioning contrast per AI-endpoint-canonical); track their "Ask Bluefin" RAG-over-docs as a reference for MiOS Portal's knowledge surface.

3. **[LocalAGI + LocalAI (mudler)](https://github.com/mudler/LocalAGI)** — MIT. *What's new:* LocalAGI v2.9.0 (2026-05-08); a **drop-in replacement for OpenAI's Responses API** (not just chat-completions), embedding LocalAI for inference + LocalRecall for RAG, with CPU/CUDA/SYCL/AMD Docker profiles, custom interpreted-Go actions, and MCP support ([mudler/LocalAGI](https://github.com/mudler/LocalAGI)). **MiOS action — reference:** a candidate sidecar backend or benchmark for the self-hosted /v1 plane, and proof the market now expects the Responses API + MCP surface; its interpreted-Go-actions model echoes MiOS's own eval-on-agent-args verbs (TD worth studying).

4. **[Harbor (av/harbor)](https://github.com/av/harbor)** — Apache-2.0. *What's new:* v0.5.3 (following v0.5.2, 2026-06-14); one-command local LLM stack with backends (Ollama, llama.cpp default since v0.5.0, vLLM, SGLang, TabbyAPI, mistral.rs…) and sidecars (Open WebUI, SearXNG, Speaches STT/TTS, ComfyUI, Traefik) auto-wired over OpenAI-compatible endpoints via Docker-Compose ([av/harbor](https://github.com/av/harbor)). **MiOS action — reference:** a strong catalog for WHICH sidecars to bundle as Quadlets and how to auto-wire /v1 endpoints — but Harbor stops at Docker-Compose; MiOS's differentiator is immutable bootc/BIB delivery + Ventoy USB + SSOT-driven wiring Harbor lacks.

## Desktop shell / liquid-glass (Wayland)

1. **[Quickshell](https://github.com/quickshell-mirror/quickshell)** — LGPL-3.0 (with GPL-3.0 components). *What's new:* v0.3.0 (2026-05-04); a QtQuick/QML toolkit for Wayland+X11 shells with composable declarative widgets (bars, tray, audio mixer, PipeWire), now the de-facto engine under Caelestia, DankMaterialShell and end-4 dots ([quickshell-mirror/quickshell](https://github.com/quickshell-mirror/quickshell)). **MiOS action — adopt_now:** the most credible engine for MiOS's Claude-lane liquid-glass shell; because the UI is declarative QML, bar layout/colors/effect params can be projected from `mios.toml [colors]` via mios-theme-render/dotfiles-registry — add a `[dotfiles.registry.quickshell]` surface, drift-gate it, and package the shell into the bootc/BIB image as a HOME artifact refreshed by mios-sync-theme.

2. **[DankMaterialShell](https://github.com/AvengeMedia/DankMaterialShell)** — MIT. *What's new:* v1.5.1 (2026-07-15); Quickshell/QML frontend + Go backend (system services, CLI, IPC) replacing waybar/swaylock/swayidle/mako/fuzzel/polkit, with wallpaper-based Material You theming propagated across GTK/Qt/terminals via matugen + dank16; runs on niri/Hyprland/Sway/labwc/Scroll/MiracleWM ([AvengeMedia/DankMaterialShell](https://github.com/AvengeMedia/DankMaterialShell)). **MiOS action — watch:** closest architectural cousin — a Go backend + CLI + IPC maps onto MiOS's agent-pipe/verb-backend pattern, wallpaper-to-system theming is the same many-surface projection MiOS drift-gates, MIT is safe to vendor/fork, and multi-compositor support hedges against Hyprland lock-in. Track its matugen plumbing as prior art for `[colors]` fan-out.

3. **[HyprGlass](https://github.com/hyprnux/hyprglass)** — BSD-3-Clause. *What's new:* v0.6.4 (2026-06-12); a Hyprland decoration-layer plugin running transparent windows through a multi-pass GPU pipeline — frosted Gaussian blur, SDF glass-slab refraction, chromatic aberration, dome-lens distortion, Fresnel edge glow — configured in Lua (Hyprland 0.55+) with per-window overrides ([hyprnux/hyprglass](https://github.com/hyprnux/hyprglass)). **MiOS action — watch:** closest literal match to the Apple-liquid-glass north star; its per-window effect params are exactly the values to drive from `mios.toml [effects]/[colors]` into the projected hyprland.conf (drift-gated like btop), and BSD-3 makes vendoring into the bootc image safe — but still v0.x/niche, so track rather than hard-depend.

4. **[Caelestia Shell](https://github.com/caelestia-dots/shell)** — GPL-3.0. *What's new:* v2.2.0 (2026-07-16); a Quickshell fluid/morphing Hyprland shell (~2,500 commits, 10.6k stars) driven entirely from a single `~/.config/caelestia/shell.json`, with glass via Hyprland blur layer_rules + wallpaper-derived dynamic color ([caelestia-dots/shell](https://github.com/caelestia-dots/shell)). **MiOS action — reference:** a working reference for the operator-defined model — `shell.json` is precisely the hand-maintained divergent config MiOS's SSOT doctrine replaces; learn its schema, then generate the config as a projected drift-gated dotfiles-registry surface rather than forking the GPL shell.
