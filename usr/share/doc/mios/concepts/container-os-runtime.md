# AI-hint: System concepts documentation for the MiOS Container-OS Runtime Architecture.
# MiOS Container-OS Runtime Architecture (mid-2026) — FINAL

## 1. Executive summary

The spine is a **Fedora bootc** immutable host built FROM **ublue-os/ucore-hci**, conditionally `-nvidia`/`-nvidia-lts` ([github.com/ublue-os/ucore](https://github.com/ublue-os/ucore)), delivered and updated as a single composefs-sealed OCI artifact by **bootc** v1.16.4 ([github.com/bootc-dev/bootc](https://github.com/bootc-dev/bootc)): `/usr` is read-only and fs-verity-sealed, `/etc` is a transient overlay over baked `/usr/etc`, and — corrected from the draft — **`/var` PERSISTS by default** on bootc/ostree ([containers/bootc storage docs](https://github.com/bootc-dev/bootc/blob/main/docs/src/filesystem.md)), which is what makes MiOS's LUKS-backed model caches, qcow2 VM disks, and ceph/gluster/stratis state coherent. Confidentiality — absent from the draft — comes from **TPM2-measured boot + LUKS FDE bound to PCR policy** via `systemd-cryptenroll`/**clevis** ([github.com/latchset/clevis](https://github.com/latchset/clevis), [tpm2-software/tpm2-tools](https://github.com/tpm2-software/tpm2-tools)); composefs supplies integrity, not encryption.

Containers are orchestrated with **no cluster control plane by default**: **Podman 6.0 + Quadlet** ([github.com/containers/podman](https://github.com/containers/podman)) bakes the 26+ mios.toml-projected `.container/.pod/.network/.volume` units read-only into `/usr/share/containers/systemd`, and systemd is the control loop. Where a real Kubernetes API is genuinely needed MiOS uses **image-mode k3s** — corrected from the draft's blanket ban, because mios.toml ships `k3s`+`k3s-selinux` and enables `mios-k3s`, and k3s runs on composefs when the SELinux policy is built in and the imperative `get.k3s.io` installer is never used ([github.com/k3s-io/k3s](https://github.com/k3s-io/k3s), [k3s SELinux](https://github.com/k3s-io/k3s-selinux)).

The Secure Boot story is the **full UKI chain**, not just kernel-module MOK: **shim → systemd-boot → signed UKI** built with **systemd-ukify** + **sbsigntools**, updated by **bootupd** ([github.com/coreos/bootupd](https://github.com/coreos/bootupd)) — and every boot-critical karg (IOMMU, hugepages, isolcpus, `vfio-pci.ids=`, `nouveau-blacklist`, `nvidia-drm.modeset=1`) is **baked and signed INTO the UKI**, so kargs are an SSOT-projected, drift-gated surface that changes only by rebuild+re-sign ([systemd UKI](https://uapi-group.org/specifications/specs/unified_kernel_image/), [ukify](https://www.freedesktop.org/software/systemd/man/latest/ukify.html)). Auto-updating an immutable host is made safe by **greenboot** health gating driving automatic bootc rollback ([github.com/fedora-iot/greenboot](https://github.com/fedora-iot/greenboot)), wired to the MiOS drift-gates and :8640 Portal.

VMs ride socket-activated **libvirt** ([gitlab.com/libvirt/libvirt](https://gitlab.com/libvirt/libvirt)), domain XML projected from the same SSOT, giving Windows/Xbox/Linux KVM guests (swtpm + OVMF Secure Boot + virtio-win + `<hyperv>` enlightenments + vfio-pci GPU). GPU is **multi-vendor**, not NVIDIA-only: **CDI** ([github.com/cncf-tags/container-device-interface](https://github.com/cncf-tags/container-device-interface)) fronts NVIDIA Container Toolkit, AMD **ROCm** (`/dev/kfd`+`/dev/dri`), Intel **oneAPI/level-zero**, and a CPU-only llama.cpp fallback, projected per detected vendor ([ROCm](https://github.com/ROCm/ROCm), [level-zero](https://github.com/oneapi-src/level-zero)). Single-GPU contention — the draft's biggest open hole — is answered concretely by **mdevctl**-provisioned mediated devices/vGPU where the silicon supports it and an explicit boot-time bind state machine where it does not ([github.com/mdevctl/mdevctl](https://github.com/mdevctl/mdevctl)). Per-unit hardware isolation remains **crun+krun over libkrun** ([github.com/libkrun/libkrun](https://github.com/libkrun/libkrun)) — but the draft's "GPU-capable microVM for CUDA inference" claim is corrected: venus is a Vulkan/graphics transport, so CUDA inside a microVM requires real vfio passthrough, not venus. Capability that cannot wait for a rebuild arrives as signed **systemd-sysext/confext** overlays; mutable userspace lives in rootless **Distrobox** — never layered into the sealed root.

## 2. Layer-by-layer architecture

### Host confidentiality: TPM2, measured boot, LUKS *(new — critic gap)*
**Choice: LUKS2 FDE bound to TPM2 PCR policy via `systemd-cryptenroll`/clevis.** composefs seals integrity but gives **no confidentiality** ([composefs](https://github.com/composefs/composefs)); model caches, VM disks, and secrets on a physical MiOS-Cat box need encryption at rest. `systemd-cryptenroll --tpm2-device=auto --tpm2-pcrs=7+11` seals the LUKS key to firmware+UKI PCRs so the disk unlocks unattended only if the measured boot chain is intact ([systemd-cryptenroll](https://www.freedesktop.org/software/systemd/man/latest/systemd-cryptenroll.html)); **clevis** ([github.com/latchset/clevis](https://github.com/latchset/clevis)) is the packaged pin alternative (TPM2 or network Tang). **HOW into MiOS:** a `[luks]`/`[tpm]` SSOT surface projects PCR policy and enrollment; `tpm2-tss`/`tpm2-tools` ([github.com/tpm2-software/tpm2-tools](https://github.com/tpm2-software/tpm2-tools)) provide attestation primitives. **GOTCHA:** PCR-7/11 policy breaks on **firmware update (fwupd/LVFS)** and on any UKI re-sign — sealed keys must carry a **recovery passphrase** (mios.toml `[auth].luks_passphrase`, never tracked) and be re-enrolled after PCR-changing events ([fwupd](https://github.com/fwupd/fwupd)). Early unlock happens in the **dracut**-built initramfs ([github.com/dracutdevs/dracut](https://github.com/dracutdevs/dracut)).

### Secure Boot signing chain: UKI, systemd-boot, bootupd *(new — critic gap; corrects MOK conflation)*
**Choice: shim → systemd-boot → signed UKI, built with ukify+sbsigntools, managed by bootupd.** The draft conflated **MOK** (kernel-*module* signing) with the whole chain. MiOS ships `systemd-boot-unsigned` + `systemd-ukify` + `sbsigntools` + `bootupd`: the kernel, initramfs, and **kargs** are fused into a **Unified Kernel Image** and signed, so kargs are immutable at runtime and changing them means **rebuild + re-sign the UKI** ([UKI spec](https://uapi-group.org/specifications/specs/unified_kernel_image/), [ukify](https://www.freedesktop.org/software/systemd/man/latest/ukify.html)). **bootupd** ([github.com/coreos/bootupd](https://github.com/coreos/bootupd)) atomically updates the ESP bootloader across the fleet. **HOW into MiOS:** re-signing with MiOS's own keys means **owning shim/db enrollment across heterogeneous MiOS-Cat firmware** (first-boot MOK/db enroll); staying on ublue's keys means trusting ublue's chain. **GOTCHA:** module-signing MOK ≠ bootloader/kernel signing — both must be reasoned about; sd-boot itself must be signed/enrolled, which the draft never accounted for.

### Kernel command line as SSOT surface *(new — critic gap)*
**Choice: a first-class `[kargs]` SSOT surface, projected + drift-gated + UKI-signed.** IOMMU (`intel_iommu=on`/`amd_iommu=on iommu=pt`), hugepages (`default_hugepagesz=1G hugepagesz=1G hugepages=N`), latency isolation (`isolcpus=`/`nohz_full=`/`rcu_nocbs=` for vLLM + VM vCPUs), `vfio-pci.ids=`, `rd.driver.pre=vfio-pci`, `nouveau-blacklist`, and `nvidia-drm.modeset=1` are all load-bearing and all live in the signed UKI ([kernel-parameters](https://www.kernel.org/doc/html/latest/admin-guide/kernel-parameters.html), [VFIO docs](https://docs.kernel.org/driver-api/vfio.html)). **HOW into MiOS:** mios.toml `[kargs]` renders the UKI cmdline at build; drift-gate diffs the signed UKI's embedded cmdline, not a mutable `/proc/cmdline` expectation. **GOTCHA:** "bake kargs at build" = rebuild+re-sign, and PCR changes re-break TPM-sealed LUKS (see above).

### Containers / Quadlet
**Choice: Podman 6.0 + Quadlet generator** ([github.com/containers/podman](https://github.com/containers/podman), Apache-2.0). The `/usr/libexec/podman/quadlet` generator emits real `.service` units at boot and on `daemon-reload`, so `After=/Requires=/BindsTo=` ordering is native systemd. **WHY:** no daemon/apiserver — kernel + systemd is the control loop; units are plain text bakeable into RO `/usr`. **HOW into MiOS:** SSOT projects each of the 26+ units into `/usr/share/containers/systemd`; rootless per-operator units use the `.../users[/${UID}]` search paths; drift-gates 25–28 diff the *generated* `.service` output. **CORRECTION (critic):** `Notify=healthy` ordering **requires the image to implement sd-notify or a defined Podman `HealthCmd=`/HEALTHCHECK** — upstream vLLM/llama.cpp images ship neither, so MiOS must inject an explicit `HealthCmd=` (HTTP probe of `/health` on :8642) or "Hermes starts After= vLLM healthy" silently degrades to start-order-only ([podman healthcheck](https://docs.podman.io/en/latest/markdown/podman-healthcheck-run.1.html)). **GOTCHA:** the 6.0 per-app-subdirectory move and CNI→Netavark removal are BREAKING — baked Podman and projector must be version-locked or units mis-resolve.

### Secrets & credentials *(new — critic gap)*
**Choice: `podman secret` / Quadlet `Secret=` for containers; `LoadCredentialEncrypted=`/sd-creds for host units; TPM-sealed at rest.** On an immutable host with transient `/etc` you cannot hand-place plaintext secrets. Podman secrets (Quadlet `Secret=mytoken,type=env`) inject API keys/model tokens into Quadlets ([podman secret](https://docs.podman.io/en/latest/markdown/podman-secret.1.html)); systemd `LoadCredentialEncrypted=` decrypts TPM/host-key-sealed credentials into a unit's private `/run` tmpfs ([systemd credentials](https://systemd.io/CREDENTIALS/)). Registry **PullSecret** for LBI/runtime pulls is projected the same way. **HOW into MiOS:** mios.toml `[auth]` holds `luks_passphrase`/`github_pat` (never tracked); a secrets projector seals them with `systemd-creds encrypt` (TPM-bound) and wires them to Quadlets/VMs — the mechanism the draft entirely lacked. **GOTCHA:** the chicken-and-egg — Portal :8640 needs secrets before SSOT is fully applied (see First-boot).

### Pods & orchestration on one host *(corrected: k3s)*
**Choice: Quadlet `.pod` units — no cluster control plane by default** ([podman](https://github.com/containers/podman)). `.pod` units (`Label=`/`ExitPolicy=`) group the GPU AI plane with sidecars as one dependency-ordered pod. **CORRECTION (critic):** containers in a `.pod` share one netns (localhost), so the draft's "monitoring sidecar can't reach the AI pod under strict isolation" applies only to **cross-pod** scraping, not intra-pod. **Kubernetes when actually needed:** MiOS ships `k3s`+`k3s-selinux` and **enables `mios-k3s`**, so the draft's flat "never k3s / unsupported on composefs" is **wrong and self-contradictory** — image-mode k3s works when the SELinux policy is built into `/usr` and the `get.k3s.io` installer is never invoked ([k3s](https://github.com/k3s-io/k3s), [k3s-selinux](https://github.com/k3s-io/k3s-selinux)). **MicroShift** 4.22 ([github.com/openshift/microshift](https://github.com/openshift/microshift)) remains the CRD/operator alternative, image-mode only. **GOTCHA:** k3s/MicroShift bring a second runtime (CRI-O/containerd) + CNI and write etcd/kine state under `/var/lib` — persistent by design on bootc, but a distinct storage owner to drift-gate.

### libvirt VMs
**Choice: libvirt** ([gitlab.com/libvirt/libvirt](https://gitlab.com/libvirt/libvirt), LGPL-2.1+, v12.5.0), socket-activated modular daemons baked via ucore-hci; nftables backend. **HOW into MiOS:** `virt-install`/`virt-xml` ([github.com/virt-manager/virt-manager](https://github.com/virt-manager/virt-manager)) render `--print-xml` from SSOT; `cockpit-machines` ([github.com/cockpit-project/cockpit-machines](https://github.com/cockpit-project/cockpit-machines)) is the read-mostly console beside :8640. **CORRECTION (critic):** domain XML and pools live under `/etc/libvirt` (transient overlay) and `/var/lib/libvirt` (**persistent**, not volatile) — so VM definitions are projected+re-applied idempotently from SSOT, and human-made cockpit VMs drift-gate red until round-tripped. **CORRECTION (critic):** GPU passthrough needs **system-mode** libvirtd — NOT rootless.

### Windows / Xbox guests *(expanded — critic)*
**Choice: the QEMU/KVM firmware+driver quartet, plus the `<hyperv>` enlightenment block the draft omitted.** **swtpm** v0.10.1 ([github.com/stefanberger/swtpm](https://github.com/stefanberger/swtpm)) gives Windows 11's TPM 2.0 via `<tpm model='tpm-crb'>`; **edk2/OVMF** ([github.com/tianocore/edk2](https://github.com/tianocore/edk2)) provides guest Secure Boot via `OVMF_CODE.secboot.fd` + per-VM VARS from `OVMF_VARS_4M.ms.fd`; **virtio-win** 0.1.271 ([github.com/virtio-win/kvm-guest-drivers-windows](https://github.com/virtio-win/kvm-guest-drivers-windows)) supplies WHQL-signed paravirtual drivers; **QEMU** 10.1 + vfio-pci hands over a real GPU. **NEW — mandatory `<hyperv>` enlightenments** (`relaxed/vapic/spinlocks/vpindex/synic/stimer/reset/frequencies/vendor_id`) or Windows 11 guests are slow and BSOD-prone; **`vendor_id` spoof + `kvm hidden state`** defeat NVIDIA **Code-43** on consumer GPUs ([libvirt formatdomain hyperv](https://libvirt.org/formatdomain.html#hypervisor-features)). **NEW:** nested-virt exposure (`<cpu mode='host-passthrough'>` + `ds/svm` or similar options for Virtualization Based Security); `<memoryBacking><hugepages/>`; `<cputune>` vcpupin + NUMA/GPU affinity; `<watchdog>`; evdev/USB input passthrough and **audio** (virtio-sound / Scream / PipeWire) for single-seat gaming. **Xbox pipeline (critic):** MiOS-XBOX is a **customized Windows image** (autounattend FSE, DeviceForm spoof, Gaming Services hydration, WSL daemon) with an **aarch64 `mios-xbox-arm`** variant — so the guest plane must wire the **virt-v2v + libguestfs + virt-firmware** ISO build/convert/SB-key-enroll pipeline ([github.com/libguestfs/virt-v2v](https://github.com/libguestfs/virt-v2v), [gitlab.com/kraxel/virt-firmware](https://gitlab.com/kraxel/virt-firmware)), including ARM-Windows guest + ARM GPU passthrough as the hard case. **GOTCHA:** NVRAM+swtpm state under svirt-labeled libvirt storage (persistent `/var`); use 4M CODE with 4M VARS consistently.

### GPU: multi-vendor plane *(rewritten — critic; NVIDIA-only was wrong)*
**Choice: CDI as the SSOT abstraction across NVIDIA / AMD / Intel / CPU-fallback.** CDI ([github.com/cncf-tags/container-device-interface](https://github.com/cncf-tags/container-device-interface)) yields vendor-neutral device tokens. **NVIDIA:** Container Toolkit v1.19.1 ([github.com/NVIDIA/nvidia-container-toolkit](https://github.com/NVIDIA/nvidia-container-toolkit)) + `nvidia-open` 610.x akmod ([github.com/NVIDIA/open-gpu-kernel-modules](https://github.com/NVIDIA/open-gpu-kernel-modules), [github.com/ublue-os/akmods](https://github.com/ublue-os/akmods)). **AMD:** **ROCm** with CDI exposing `/dev/kfd`+`/dev/dri`, `amdgpu` kmod, vLLM/llama.cpp on ROCm ([github.com/ROCm/ROCm](https://github.com/ROCm/ROCm)). **Intel:** oneAPI/**level-zero** + `xe`/`i915`, SYCL llama.cpp ([github.com/oneapi-src/level-zero](https://github.com/oneapi-src/level-zero)). **CPU-only box:** llama.cpp CPU/Vulkan fallback so the /v1 plane always exists. **CORRECTION (critic):** the draft recommended JIT `nvidia-cdi-refresh` writing `/etc/cdi` — **fragile on bootc** because `/etc` resets toward baked `/usr/etc`; the robust pattern is **build-time CDI into `/usr/share/cdi`** or **confext-projected**, not runtime JIT ([nvidia-ctk cdi](https://github.com/NVIDIA/nvidia-container-toolkit)). **HOW into MiOS:** mios.toml conditionally projects `gpu-nvidia`/`gpu-amd-compute`/`gpu-intel-compute`/`gpu-mesa`/`gpu-cdi-toolkits` per detected vendor. **GOTCHA:** akmod RPM is kernel-version-locked (base image + driver move together — the single biggest immutable coupling); every out-of-tree module needs **MOK-enrolled signing** or kernel lockdown refuses it under Secure Boot.

### GPU ownership: boot-time bind, vfio, mdev *(new state machine — resolves draft open-Q #1)*
**Choice: an SSOT `[gpu]` state machine, whole-device passthrough; mediated vGPU out of scope.** **CORRECTION (critic):** the draft hand-waved "single-GPU = container XOR VM" without a bind mechanism and ignored **mdevctl** — which MiOS packages and which is exactly the vGPU/GVT-g/NVIDIA-vGPU answer to concurrency ([github.com/mdevctl/mdevctl](https://github.com/mdevctl/mdevctl)). We reject mediated vGPU splits on the driver-free host: (1) **whole-device passthrough** via `vfio-pci` (the default mode; mediated vGPU is out of scope); (2) **container-plane** (akmod driver owns the GPU, CDI); (3) **vfio-passthrough** (`vfio-pci.ids=` + `rd.driver.pre=vfio-pci` bound early in initramfs, `force_drivers`). **Heterogeneous-HW landmines (critic):** GPUs often share an **IOMMU group** with other PCIe functions; the **ACS-override patch is UNSIGNED** and breaks under lockdown/Secure Boot — so MiOS must select hardware with clean groups rather than patch; pass the GPU's **multifunction HDMI-audio** device together; handle **AMD reset bug** / NVIDIA reset. **CRITICAL CONFLICT (critic):** vfio claiming the single GPU **blanks the host console and the liquid-glass Hyprland shell** — the desktop shell and a passthrough Windows guest cannot both own one GPU; the state machine must encode this exclusivity, and **rebinding is imperative (driverctl) = anti-immutable**, so the honest MiOS posture is boot-time selection (rebuild/reboot to switch) except where mdev provides true concurrency.

### MicroVM isolation *(corrected: venus ≠ CUDA)*
**Choice: crun's krun handler over libkrun** (crun [github.com/containers/crun](https://github.com/containers/crun); libkrun-1.19.4 [github.com/libkrun/libkrun](https://github.com/libkrun/libkrun)). `Runtime=krun` gives a Quadlet a hard KVM boundary by swapping one line; libkrun's SEV-SNP/TDX variants are the confidential-inference on-ramp. **CORRECTION (critic):** the draft implied krun gives isolated **CUDA** inference via "virtio-gpu venus" — **venus is a Vulkan/graphics transport, not CUDA**; running vLLM (CUDA) in a microVM requires **real vfio GPU passthrough into the microVM**, which venus does not provide. So `Runtime=krun` is the isolation lever for CPU/untrusted work and Vulkan/SYCL inference; CUDA-in-microVM is a separate vfio path. **HOW into MiOS:** projected units carry `Runtime=krun` only where a hard boundary is wanted; `.so` pinned as SBOM (1.19.x — 2.0 is an ABI break). **GOTCHA:** needs `/dev/kvm` + nested virt — untestable on the current MiOS-DEV VM. **Watch:** `crun-vm` ([github.com/containers/crun-vm](https://github.com/containers/crun-vm)), Cloud Hypervisor v53 ([github.com/cloud-hypervisor/cloud-hypervisor](https://github.com/cloud-hypervisor/cloud-hypervisor)).

### sysext / confext layering
**Choice: systemd-sysext/confext** ([github.com/systemd/systemd](https://github.com/systemd/systemd), v257+), overlaying `/usr`/`/opt` (confext onto `/etc`), gated by `extension-release.NAME` matching host os-release. **WHY:** the only Law-2-compatible add-capability-without-rebuild path. **HOW into MiOS:** ship extra CLIs, vLLM/llama.cpp binaries, driver stacks, or **build-time CDI/SELinux policy** as signed sysext/confext keyed to MiOS os-release; a `[sysext]` surface drift-gated like Quadlets. **GOTCHA:** confext→`/etc` races bootc's transient `/etc` overlay (reason about ordering); kernel-module sysexts still require MOK-signed modules — sysext does NOT bypass signature enforcement. **Reference:** flatcar/sysext-bakery ([github.com/flatcar/sysext-bakery](https://github.com/flatcar/sysext-bakery)), travier/sysexts-manager ([github.com/travier/sysexts-manager](https://github.com/travier/sysexts-manager)) — both need self-hosted repos for local-first.

### Storage & persistence *(expanded — critic; volatile-/var premise removed)*
**Choice: composefs + additionalimagestores for the sealed root; ceph/gluster/stratis/lvm/mdadm for persistent `/var`.** **CORRECTION (critic):** the draft's whole "volatile tmpfs `/var` each boot" premise is **wrong** — bootc/ostree **persists `/var` by default** ([bootc filesystem](https://github.com/bootc-dev/bootc/blob/main/docs/src/filesystem.md)), and MiOS packages a full persistent-storage answer the draft left "unspecified": **ceph/cephadm** (opt-in `mios-ceph` Quadlet needing `/etc/ceph/ceph.conf`, [github.com/ceph/ceph](https://github.com/ceph/ceph)), **glusterfs** ([github.com/gluster/glusterfs](https://github.com/gluster/glusterfs)), **stratis** ([github.com/stratis-storage/stratisd](https://github.com/stratis-storage/stratisd)), **lvm2**, **mdadm**. composefs v1.0.8 ([github.com/composefs/composefs](https://github.com/composefs/composefs)) + containers/storage `additionalimagestores` ([github.com/containers/storage](https://github.com/containers/storage)) ship the 26+ images as a RO baked store so Quadlets start with no first-boot pull. **HOW into MiOS:** named volumes, qcow2 VM disks, and the multi-GB model cache live under persistent `/var/lib` on a LUKS-encrypted, optionally ceph/gluster-backed pool projected from an SSOT storage surface; `:z/:Z` SELinux relabel on shared persistent volumes. **GOTCHA:** DR/rollback state compat (below).

### Networking *(corrected: pasta scope)*
**Choice: netavark + aardvark-dns + pasta for containers; bridge/macvtap for passthrough VMs.** netavark v2.0.0 ([github.com/containers/netavark](https://github.com/containers/netavark), nftables-only, strict bridge isolation default) materializes `[network]` into bridges; aardvark-dns v2.0.0 ([github.com/containers/aardvark-dns](https://github.com/containers/aardvark-dns)) keeps `MIOS_AI_ENDPOINT` symbolic; **pasta** ([passt.top](https://passt.top/passt/about/)) serves rootless Podman. **CORRECTION (critic):** the draft's "pasta is the one transport for both Podman and libvirt/KVM" **overreaches** — system-mode passthrough VMs (required for GPU) typically use **bridge/macvtap**, and libvirt passt is a specific newer backend, not a unified default; since system-mode libvirtd is not rootless, "one transport both lanes" is inconsistent. **Firewall SSOT:** host firewalld, netavark, and libvirt-nftables are three potential nftables writers — one SSOT arbiter owns the table (open-Q).

### Observability, resources & rootless plumbing *(new — critic)*
**Choice: persistent journald + PCP + cgroupv2/systemd-oomd/zram, with lingering reconciled to persistent `/var`.** With `/var` persistent, `/var/log/journal` **survives** (draft's "logs vanish" fear was tied to the wrong volatile premise), enabling retention, `sos` support bundles, and **PCP** metrics into :8640 ([github.com/performancecopilot/pcp](https://github.com/performancecopilot/pcp), cockpit-pcp). **Resource arbitration** for a box running a huge Windows VM + vLLM: cgroupv2 `MemoryMax=`/`CPUQuota=` per Quadlet, **systemd-oomd** pressure-based kills, **zram** swap ([github.com/systemd/zram-generator](https://github.com/systemd/zram-generator)). **Rootless plumbing (critic):** subuid/subgid projected into transient `/etc`; **`loginctl enable-linger`** writes `/var/lib/systemd/linger` — persistent on bootc, so rootless user Quadlets survive logout (this only conflicted under the wrong volatile-/var premise).

### Container image trust & registries *(new — critic)*
**Choice: sigstore/cosign enforcement via policy.json + local-first mirror.** `/etc/containers/policy.json` sigstore verification + `registries.d` enforce signatures on LBI/runtime-pulled AI images ([containers/image policy](https://github.com/containers/image/blob/main/docs/containers-policy.json.5.md), [github.com/sigstore/cosign](https://github.com/sigstore/cosign)); short-name aliases pin unqualified searches. **Air-gap/local-first:** a **zot** or Forgejo pull-through mirror ([github.com/project-zot/zot](https://github.com/project-zot/zot)) serves the fleet — a glaring hole the draft omitted given "everything drift-gated."

### Identity, security & power *(new — critic)*
**Choice: FreeIPA identity, CrowdSec IPS, NUT UPS — all packaged, all SSOT-wired.** **FreeIPA** ([github.com/freeipa/freeipa](https://github.com/freeipa/freeipa)) provides fleet auth + Windows-guest Kerberos/domain join; **CrowdSec** ([github.com/crowdsecurity/crowdsec](https://github.com/crowdsecurity/crowdsec)) + crowdsec-dashboard Quadlet is the IPS; **NUT** ([github.com/networkupstools/nut](https://github.com/networkupstools/nut)) triggers graceful shutdown of VMs+inference on power loss. **chrony** ([chrony.tuxfamily.org](https://chrony.tuxfamily.org/)) — required for TPM attestation, TLS validity, Kerberos, and VM clocks — was omitted by the draft and is mandatory here.

### Remote display & host↔guest sharing *(new — critic; corrects "SMB-less")*
**Choice: SSOT-wired Guacamole/SPICE for display; Samba/virtiofsd for shares.** MiOS ships `mios-guacamole`/guacd (browser RDP, [github.com/apache/guacamole-server](https://github.com/apache/guacamole-server)) + `spice-vdagent`/`spice-webdavd` as the wired alternative to "watch"-only Looking Glass ([github.com/gnif/LookingGlass](https://github.com/gnif/LookingGlass)). **CORRECTION (critic):** the draft's "SMB-less, virtiofsd+WinFsp" contradicts packaged **samba/cifs-utils/freerdp**; MiOS projects a share surface with **both** virtiofsd ([gitlab.com/virtio-fs/virtiofsd](https://gitlab.com/virtio-fs/virtiofsd)) for Linux guests and SMB for Windows.

### bootc ↔ Podman binding, greenboot, DR
**Choice: bootc + LBI + osbuild image-builder + bcvk, gated by greenboot, with DR state-version guards.** bootc v1.16.4 is the atomic OCI update/rollback engine; **LBI** symlinks a `.image` into `/usr/lib/bootc/bound-images.d` to pull the AI image before reboot ([bootc](https://github.com/bootc-dev/bootc)). **osbuild image-builder `--bootc-ref`** ([github.com/osbuild/bootc-image-builder](https://github.com/osbuild/bootc-image-builder)) produces the MiOS-Cat ISO/raw; **bcvk** v0.18.0 ([github.com/bootc-dev/bcvk](https://github.com/bootc-dev/bcvk)) is the ephemeral-VM test loop. **greenboot (new — critic):** `greenboot` ([github.com/fedora-iot/greenboot](https://github.com/fedora-iot/greenboot)) runs health checks (GPU driver up, :8640 up, key Quadlets healthy, drift-gates green) and **auto-rolls-back a bad bump** — the single most important auto-update primitive, wired to MiOS drift-gates/Portal. **DR (new — critic):** rolling the OS image back while persistent `/var` (qcow2 formats, model cache, DB schemas) moved forward risks incompatibility — MiOS needs restic/snapshot backup + a **state-version guard** across bootc rollback ([github.com/restic/restic](https://github.com/restic/restic)). **GOTCHA:** LBI is rootful-only, `Image=`-only, no Arch/PullSecret — multi-arch + private-registry solved out-of-band.

### First-boot provisioning & fleet enrollment *(new — critic)*
**Choice: cloud-init/Ignition + pxe-hub/iPXE for per-machine identity.** A heterogeneous MiOS-Cat box gets per-machine identity, SSOT, LUKS passphrase, and MOK/db enrollment on first boot via cloud-init/Ignition ([github.com/coreos/ignition](https://github.com/coreos/ignition)); **mios-pxe-hub/iPXE** ([github.com/ipxe/ipxe](https://github.com/ipxe/ipxe)) network-boots the fleet. Resolves the secrets chicken-and-egg: Portal :8640 gets its bootstrap secret from the sealed first-boot credential before full SSOT apply.

### SELinux runtime surface *(new — critic)*
**Choice: projected booleans/modules + udica, beyond `nvidia_container_t`.** `container_use_devices`, `virt_use_*`, `:z/:Z` relabel on persistent `/var`, and **udica**-generated policy for custom containers ([github.com/containers/udica](https://github.com/containers/udica)) ship via build/confext; cockpit-selinux is the console. The policy SSOT surface must be explicit, not implicit.

### distrobox / mutable dev
**Choice: Distrobox** v1.8.2.5 ([github.com/89luca89/distrobox](https://github.com/89luca89/distrobox), v2 Go RC). `distrobox assemble` renders `distrobox.ini` from a `[dotfiles.registry.<dev-box>]` surface (ADR-0010), drift-gated with `assemble --dry-run`; `distrobox-export` surfaces tools into the Quickshell menu. Dev boxes are a SEPARATE user surface, not in the 26+ fleet. **GOTCHA:** exports land on persistent `/var/home`; `--nvidia` in-box CUDA must match the signed host kmod; `assemble` needs `replace=true` to converge. **Watch:** Toolbx ([github.com/containers/toolbox](https://github.com/containers/toolbox)).
